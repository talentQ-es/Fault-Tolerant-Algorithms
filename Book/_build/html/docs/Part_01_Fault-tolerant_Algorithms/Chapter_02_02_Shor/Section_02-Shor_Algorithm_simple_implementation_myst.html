El algoritmo de Shor se basa en el algoritmo de Estimaci√≥n de Fase Cu√°ntica (Quantum Phase Estimation) ([3] y [4]), que a su vez usa la Transformada de Fourier Cu√°ntica (Quantum Fourier Transform) ([2] y [3]). El algoritmo de Shor lo que hace es convertir el problema de la factorizaci√≥n de un n√∫mero en un problema de encontrar el periodo de una funci√≥n, el cual puede ser implementado en un tiempo polin√≥mico.

Para factorizar un n√∫mero \(N\) b√°sicamente el algoritmo de Shor lo que hace calcular el periodo de una funci√≥n (peri√≥dica) de la forma

(3)#\[\begin{equation} f(x) = a^x \text{mod} N \end{equation}\]

donde \(a\) y \(N\) son enteros positivos mayores que 1, siendo adem√°s \(a\) < \(N\) y no teniendo factores comunes. La operaci√≥n (\(z\) mod\(N\)) a lo que se refiere es a quedarnos con el resto de dividir el n√∫mero que \(z\) por \(N\). El periodo \(r\) de esta funci√≥n se calcula mediante el algoritmo de estimaci√≥n de fase cu√°ntica. Una vez se tiene el periodo \(r\), si este es par (sino hay que probar con otro valor de \(a\)) se pueden calcular los factores de \(N\) ya que existe una alta probabilidad de que el m√°ximo com√∫n divisor de \(N\) y \(a^{\,r/2}-1\) o \(a^{\,r/2}+1\) sea un factor propio de \(N\).

En esta gu√≠a vamos a seguir la explicaci√≥n del Notebook de Qiskit [5].

2.1 - Hallar del periodo de una funci√≥n (Period Finding)#

2.1.1 - La funci√≥n#

Como comentamos en la introducci√≥n, lo que queremos es hallar el periodo de la funci√≥n

\[ f(x) = a^x \text{mod } N \]

donde \(a\) y \(N\) son enteros positivos mayores que 1, siendo adem√°s \(a\) < \(N\) y no teniendo factores comunes. La operaci√≥n (\(z\) mod\(N\)) a lo que se refiere es a quedarnos con el resto de dividir el n√∫mero que \(z\) por \(N\). A este tipo de funciones se las denomina exponenciales moduladas.

Denominaremos \(r\) al valor del periodo de la funci√≥n \(f(x)\), es decir, \(r\) es el m√≠nimo valor entero para que se cumple:

\[ f(x+r) = f(x) \]

En la siguiente figura vemos un ejemplo de este tipo de funciones con \(a = 3\) y \(N=35\). Vemos que para este caso el periodo es \(r = 12\). (V√©ase que las lineas puntuadas que unen las cruces son solo por est√©tica).

2.1.2 - Soluci√≥n: Estimaci√≥n de fase de un operador U#

El algoritmo de Shor se basa en implementar el algoritmo de estimaci√≥n de fases al operador unitario

\[ U |y \rangle \equiv | ay \, \text{mod} N \rangle \]

Al aplicar sucesivas veces el operador \(U\) sobre el estado \(|1 \rangle\) vamos obteniendo los valores de \(f(x)\) con \(x \in \mathbb{N}\), esto es,

\[ U^x |1 \rangle = | f(x) \rangle \]

Por ejemplo, para el caso que vimos en la gr√°fica anterior (\(a=3\) y \(N=35\)) tenemos

(4)#\[\begin{align} U^0 |1\rangle & = |1\rangle \\ U |1\rangle & = |3\rangle \\ U^2 |1\rangle & = |9\rangle \\ \vdots \\ U^{r-1} |1\rangle & = |12\rangle \\ U^r |1\rangle & = |1\rangle \end{align}\]

(Recordemos que dado un estado de \(n\) qubits \(\left| x \right\rangle\) tenemos que \(\left| x \right\rangle = \left| x_1 x_2 \dots x_n \right\rangle\) donde \(x_1\) es el bit m√°s significativo.)

Como podemos ver, aplicar una vez m√°s el operador \(U\) significa pasar de un n√∫mero al siguiente de la lista peri√≥dica. Ve√°moslo expl√≠citamente:

(5)#\[\begin{align} U(U^0 |1\rangle) & = U(|1\rangle) = |3\rangle \\ U(U |1\rangle) & = U(|3\rangle) = |9\rangle \\ U(U^2 |1\rangle) & = U(|9\rangle) = |27\rangle\\ \vdots \\ U(U^{r-1} |1\rangle) & = U(|12\rangle) =|1\rangle \\ U(U^r |1\rangle) & = U(|1\rangle) = |3\rangle \end{align}\]

Con esto se entiende f√°cilmente que la superposici√≥n equiprobable de todos los estados es un autoestado del operador \(U\) con autovalor 1:

\[ | u_0 \rangle = \frac{1}{\sqrt{r}} \sum^{r-1}_{k=0} |a^k \, \text{mod} N \rangle, \qquad \text{donde} \quad U|u_0 \rangle = |u_0\rangle \]

Ejemplo

Ejemplo (Subtitulo del ejercicio/ejemplo)

(6)#\[\begin{align} U |u_0 \rangle & = U \left[ \frac{1}{\sqrt{12}} \left( |1 \rangle + |3\rangle + |9 \rangle + \dots |4 \rangle + |12 \rangle \right) \right] = \\ & = \frac{1}{\sqrt{12}} \left( U|1 \rangle + U|3\rangle + U|9 \rangle + \dots U|4 \rangle + U|12 \rangle \right) = \\ & =\frac{1}{\sqrt{12}} \left( |3 \rangle + |9\rangle + |27 \rangle + \dots |12 \rangle + |1 \rangle \right) = \\ & = |u_0 \rangle \end{align}\]

Un autoestado de autovalor 1 no nos es muy interesante a la hora de aplicar el algoritmo de estimaci√≥n fase. Otro conjunto de autoestados mucho m√°s interesantes son aquellos de la forma:

\[ | u_s \rangle = \frac{1}{\sqrt{r}} \sum^{r-1}_{k=0} e^{- \boldsymbol{2 \pi i}k \frac{s}{r}} |a^k \, \text{mod} N \rangle, \qquad \text{donde} \quad U|u_s \rangle = e^{\boldsymbol{2 \pi i} \frac{s}{r}}|u_s\rangle \]

donde \(0 \leq s \leq r-1\). Si ahora aplicamos el algoritmo de estimaci√≥n de fase cu√°ntica a uno de estos autoestados \(|u_s \rangle\), lo que obtendremos en el registro de conteo es \(|2^n s/r \rangle\). De aqu√≠ podemos extraer el valor de \(r\). Sin embargo, para preparar el estado \(|u_s \rangle\) tenemos que conocer \(r\), es decir, lo que queremos calcular.

Un soluci√≥n elegante y f√°cil de implementar es darnos cuenta de que la suma de todos estos estados \(|u_s \rangle\) nos da el estado \(|1\rangle\), esto es,

\[ \frac{1}{r} \sum_{s=0}^{r-1} |u_s \rangle = \frac{1}{r} \left(|u_0 \rangle + |u_1 \rangle + \dots +|u_{r-1} \rangle \right) = | 1 \rangle \]

Si ahora aplicamos el algoritmo de estimaci√≥n de fase cu√°ntico (QPS) al estado |1> (un estado f√°cilmente implementable) obtenemos una superposici√≥n equiprobable de estados de la forma \(|2^n s/r \rangle\), es decir:

\[ |0 \rangle|1 \rangle \xrightarrow{QPS} \frac{1}{\sqrt{r}} \left( \left|2^t \frac{1}{r} \right\rangle + \left|2^t \frac{2}{r} \right\rangle + \dots + \left|2^t \frac{r-1}{r} \right\rangle \right) |1 \rangle \]

donde \(t\) es el n√∫mero de qubits del registro de conteo. Usando el las fracciones continuas [6] podemos calcular \(r\) a partir de los cocientes \(s/r\).

2.2 - Implementaci√≥n (ad hoc) en Qiskit para \(N=15\)#

En esta secci√≥n vamos a ver una implementaci√≥n un poco ad hoc del algoritmo de Shor para factorizar el n√∫mero 15. Con lo de ad hoc a lo que nos referimos es a que el or√°culo que representa la exponencial modulada del operador \(U\) est√° construido espec√≠ficamente para el caso de factorizar 15 y no se va a explicar como funciona. El objetivo de esta secci√≥n es ver como tratamos los resultados del circuito, aplicando el m√©todo de las fracciones continuas [6] para obtener \(r\) y no tanto ver como se construye el or√°culo.

En concreto, el caso que vamos a resolver es el de \(a=7\) y \(N=15\). Es f√°cil ver que para este caso tenemos un periodo de \(r = 4\), donde los cuatro posibles estados son \(| 1 \rangle\), \(| 7 \rangle\), \(| 4 \rangle\) y \(| 13 \rangle\).

Nota

Un or√°culo es, por decirlo as√≠, la forma que se usa en computaci√≥n cu√°ntica para denominar a las funciones de las cuales queremos extraer informaci√≥n haciendo llamadas a las mismas. El or√°culo es una funci√≥n que desconoces c√≥mo est√° hecha, s√≥lo sabes lo que hace. Como a los or√°culos griegos, s√≥lo te est√° permitido interrogarles, pero no preguntarles c√≥mo saben que esa es la respuesta.

Al final los or√°culos se representan como secciones del circuito que se dedica a aplicar un operador \(U\). En nuestro caso, por ejemplo, llamamos or√°culo a la funci√≥n que construye el operador de la exponencial modulada \(U^{2^j} |y \rangle = | a^{2^j}y \, \text{mod} N \rangle\).

2.2.1 - Caso con muchos shots (ineficiente pero did√°ctico)#

Como el n√∫mero elegido para la factorizaci√≥n es peque√±o (necesitamos pocos qubits) vamos a lanzar varios shots al circuito para ver todos los posibles resultado. Veremos que esto es en realidad una perdida de tiempo, pues no nos hace falta tener toda la distribuci√≥n. # Oracle: Modular Exponentiation U^{2^j} |y> = |a^{2^j}y mod N> 
def c_amod15(a, power): # It use 4 
    """Controlled multiplication by a mod 15"""
    if a not in [2,4,7,8,11,13]:
        raise ValueError("'a' must be 2,4,7,8,11 or 13")
    U = QuantumCircuit(4)        
    for iteration in range(power):
        if a in [2,13]:
            U.swap(0,1)
            U.swap(1,2)
            U.swap(2,3)
        if a in [7,8]:
            U.swap(2,3)
            U.swap(1,2)
            U.swap(0,1)
        if a in [4, 11]:
            U.swap(1,3)
            U.swap(0,2)
        if a in [7,11,13]:
            for q in range(4):
                U.x(q)
    U = U.to_gate()
    U.name = "%i^%i mod 15" % (a, power)
    c_U = U.control()
    return c_U

# Inverse Quantum Fourier Transformation of n qubits
def qft_dagger(n):
    """n-qubit QFTdagger the first n qubits in circ"""
    qc = QuantumCircuit(n)
    # Don't forget the Swaps!
    for qubit in range(n//2):
        qc.swap(qubit, n-qubit-1)
    for j in range(n):
        for m in range(j):
            qc.cp(-np.pi/float(2**(j-m)), m, j)
        qc.h(j)
    qc.name = "QFT‚Ä†"
    return qc

Nota

V√©ase que el m√°ximo com√∫n divisor se calcula usando la funci√≥n gcd() del m√≥dulo math de python. Esto es, se calcula cl√°sicamente (sin ning√∫n circuito cu√°ntico), pues como ya comentamos, el algoritmo de Euclides [7] se implementa de forma eficiente en un ordenado cl√°sico.

Ahora definimos una funci√≥n que aplique el algoritmo de estimaci√≥n de fases para fase \(s/r\) donde

\[ a^r \text{ mod} N = 1 \]

def qpe_amod15(a):
    n_count = 8
    qc = QuantumCircuit(4+n_count, n_count)
    for q in range(n_count):
        qc.h(q)     # Initialize counting qubits in state |+>
    qc.x(3+n_count) # And auxiliary register in state |1>
    for q in range(n_count): # Do controlled-U operations
        qc.append(c_amod15(a, 2**q), 
                 [q] + [i+n_count for i in range(4)])
    qc.append(qft_dagger(n_count), range(n_count)) # Do inverse-QFT
    qc.measure(range(n_count), range(n_count))
    # Simulate Results
    aer_sim = Aer.get_backend('aer_simulator')
    # Setting memory=True below allows us to see a list of each sequential reading
    t_qc = transpile(qc, aer_sim)
    result = aer_sim.run(t_qc, shots=1, memory=True).result() # One shot
    readings = result.get_memory()
    print("Register Reading: " + readings[0])
    phase = int(readings[0],2)/(2**n_count)
    print("Corresponding Phase: %f" % phase)
    return phase
